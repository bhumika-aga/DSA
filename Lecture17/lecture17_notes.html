Graphs

“A graph is nodes connected by edges. BFS finds the shortest path; DFS finds connectivity. Everything else — Dijkstra, topological sort, MST — is built on top of those two primitives.”

📚 Lecture 17 Phase 3 — Advanced DS ⏱ 12 Days 🎯 45 Problems 💼 Google · Meta · Amazon · Microsoft · Uber
🌍
Why does this topic matter?
Graphs are the single largest topic in FAANG interviews by problem count. Nearly every system can be modelled as a graph — social networks, maps, dependency resolution, state machines. The good news: 95% of all graph interview problems use exactly two primitives — BFS and DFS. Learn to recognise which one to reach for and every other algorithm (Dijkstra, Kahn's, Kruskal's) is just a variation.
🔗
How it connects: BFS is the level-order traversal you already know from trees (Lecture 15), but on a general graph with cycles you must track visited[]. Dijkstra's uses the min-heap from Lecture 16. Kruskal's MST needs Union-Find (Lecture 18 — next). After Graphs, Dynamic Programming on DAGs and interval scheduling both rely on topological order.

📈 Graph Representations

A graph G = (V, E) has V vertices (nodes) and E edges. How you store it determines the complexity of every operation.

Representation Space Check Edge List Neighbours Best For
Adjacency List O(V + E) O(degree) O(degree) Sparse graphs (most interview problems)
Adjacency Matrix O(V²) O(1) O(V) Dense graphs, Floyd-Warshall
Edge List O(E) O(E) O(E) Kruskal's MST (sort edges)

wAdj.get(u).add(new int[]{v, weight});

🌊 BFS — Breadth-First Search

BFS visits nodes level by level. Because it expands the closest nodes first, it guarantees shortest path in an unweighted graph. Use a Queue and a visited set/array.

🎯 BFS Recognition Signal

"Shortest path", "minimum steps", "minimum moves", "fewest hops" in an unweighted graph or grid → reach for BFS first. If edges have weights, switch to Dijkstra's.

Single-Source BFS Template

☕ Java · BFS Shortest Path Template 
int[] bfs(List<List<Integer>> adj, int src, int n) {
    int[] dist = new int[n];
    Arrays.fill(dist, -1);
    dist[src] = 0;
    Queue<Integer> q = new LinkedList<>();
    q.offer(src);

    while (!q.isEmpty()) {
        int node = q.poll();
        for (int nei : adj.get(node)) {
            if (dist[nei] == -1) {     // not visited
                dist[nei] = dist[node] + 1;
                q.offer(nei);
            }
        }
    }
    return dist;
}
// Time: O(V + E)  Space: O(V)
TimeO(V + E)
SpaceO(V)

Multi-Source BFS

When multiple starting nodes all exist at distance 0, seed them all into the queue at once before the BFS loop begins. This avoids running BFS from each source separately.

☕ Java · Multi-Source BFS (Rotting Oranges pattern) 
// Seed all rotten oranges at distance 0
Queue<int[]> q = new LinkedList<>();
for (int r = 0; r < rows; r++)
    for (int c = 0; c < cols; c++)
        if (grid[r][c] == 2) q.offer(new int[]{r, c, 0});  // {row, col, time}

int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
int maxTime = 0;

while (!q.isEmpty()) {
    int[] cur = q.poll();
    int r = cur[0], c = cur[1], t = cur[2];
    maxTime = Math.max(maxTime, t);
    for (int[] d : dirs) {
        int nr = r + d[0], nc = c + d[1];
        if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && grid[nr][nc] == 1) {
            grid[nr][nc] = 2;   // mark visited by overwriting
            q.offer(new int[]{nr, nc, t + 1});
        }
    }
}

📃 DFS — Depth-First Search

DFS explores as deep as possible before backtracking. Use it when you need connectivity, paths, cycles, or component labelling. Can be written recursively or iteratively (explicit stack).

🎯 DFS Recognition Signal

"Connected components", "number of islands", "can you reach X from Y", "all paths", "cycle exists" → DFS. Also: any problem where you need to explore an entire region (flood fill, island area).

DFS Template + Connected Components

☕ Java · DFS — Count Connected Components 
boolean[] visited;

int countComponents(List<List<Integer>> adj, int n) {
    visited = new boolean[n];
    int count = 0;
    for (int i = 0; i < n; i++) {
        if (!visited[i]) {
            dfs(adj, i);
            count++;
        }
    }
    return count;
}

void dfs(List<List<Integer>> adj, int node) {
    visited[node] = true;
    for (int nei : adj.get(node))
        if (!visited[nei]) dfs(adj, nei);
}
// Time: O(V + E)  Space: O(V) recursion stack
⚠
Stack overflow on large inputs: Java's default stack handles ~10k–30k recursive calls. For n > 104, convert to iterative DFS using an explicit Deque<Integer> stack.

🔁 Cycle Detection

Directed Graph — 3-Color DFS

Track three states per node: 0 = unvisited, 1 = in current DFS path (grey), 2 = fully processed (black). If during DFS you reach a grey node, you found a back edge → cycle.

☕ Java · Directed Cycle Detection (3-color) 
int[] color; // 0=white, 1=grey, 2=black

boolean hasCycle(List<List<Integer>> adj, int n) {
    color = new int[n];
    for (int i = 0; i < n; i++)
        if (color[i] == 0 && dfs(adj, i)) return true;
    return false;
}

boolean dfs(List<List<Integer>> adj, int node) {
    color[node] = 1;  // mark grey (in progress)
    for (int nei : adj.get(node)) {
        if (color[nei] == 1) return true;  // back edge = cycle
        if (color[nei] == 0 && dfs(adj, nei)) return true;
    }
    color[node] = 2;  // mark black (done)
    return false;
}

Undirected Graph — Parent-Skip DFS

In undirected graphs the edge u↔v appears in both directions. When doing DFS from u via edge to v, skip the edge back to u (the parent). If you still reach an already-visited node, it's a cycle.

☕ Java · Undirected Cycle Detection (parent-skip) 
boolean dfs(List<List<Integer>> adj, int node, int parent, boolean[] vis) {
    vis[node] = true;
    for (int nei : adj.get(node)) {
        if (!vis[nei]) {
            if (dfs(adj, nei, node, vis)) return true;
        } else if (nei != parent) {  // visited AND not the edge we came from
            return true;
        }
    }
    return false;
}
📌
BFS-based cycle detection (undirected): Also works — track parent per node in the BFS queue. But the DFS parent-skip is cleaner in interviews. For directed graphs, BFS-based is Kahn's algorithm (cycle = not all nodes processed).

▶ Topological Sort

A topological order of a DAG lists every node before all nodes it points to. Only valid on Directed Acyclic Graphs. If a cycle exists, no topological order exists. Use case: course prerequisites, build dependency order, task scheduling.

Kahn's Algorithm (BFS-based) — Recommended

💡
Intuition: A node with in-degree 0 has no dependencies — process it first. After removing it, some neighbours lose a dependency (in-degree drops). If their in-degree reaches 0, they're ready. Repeat. If the output order has fewer than V nodes, there was a cycle.
☕ Java · Kahn's Topological Sort 
int[] topoSort(List<List<Integer>> adj, int n) {
    int[] inDegree = new int[n];
    for (int u = 0; u < n; u++)
        for (int v : adj.get(u)) inDegree[v]++;

    Queue<Integer> q = new LinkedList<>();
    for (int i = 0; i < n; i++)
        if (inDegree[i] == 0) q.offer(i);

    int[] order = new int[n];
    int idx = 0;
    while (!q.isEmpty()) {
        int node = q.poll();
        order[idx++] = node;
        for (int nei : adj.get(node))
            if (--inDegree[nei] == 0) q.offer(nei);
    }
    // idx < n means cycle exists — no valid topo order
    return idx == n ? order : new int[0];
}
// Time: O(V + E)  Space: O(V)

DFS-based Topological Sort

After fully processing a node (all descendants done), push it onto a stack. Reverse the stack for the order. Equivalent to Kahn's but DFS-native — useful when you already have DFS in the solution.

☕ Java · DFS Topo Sort (post-order + stack) 
Deque<Integer> stack = new ArrayDeque<>();
boolean[] visited = new boolean[n];

void dfs(int node) {
    visited[node] = true;
    for (int nei : adj.get(node))
        if (!visited[nei]) dfs(nei);
    stack.push(node);  // push AFTER all neighbours done
}

// Call dfs(i) for each unvisited i, then read stack top-to-bottom

⚙ Dijkstra's Algorithm

Dijkstra finds the shortest path in a weighted graph with non-negative edges. It is essentially BFS with a min-heap instead of a plain queue, always processing the node with the currently known shortest distance.

🎯 Dijkstra Recognition Signal

"Shortest path", "minimum cost/time/effort" and edges have non-negative weights → Dijkstra. Weights are negative? → Bellman-Ford. K-stops constraint? → Modified Bellman-Ford (DP with relaxation limit).

☕ Java · Dijkstra's Shortest Path (min-heap) 
int[] dijkstra(List<List<int[]>> adj, int src, int n) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;

    // min-heap: {dist, node}
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    pq.offer(new int[]{0, src});

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int d = cur[0], node = cur[1];

        if (d > dist[node]) continue;  // stale entry — skip

        for (int[] edge : adj.get(node)) {
            int nei = edge[0], w = edge[1];
            if (dist[node] + w < dist[nei]) {
                dist[nei] = dist[node] + w;
                pq.offer(new int[]{dist[nei], nei});
            }
        }
    }
    return dist;
}
// Time: O((V + E) log V)  Space: O(V)
📌
The “stale entry” check is critical. Java's PriorityQueue does not support decrease-key, so when you find a shorter path you push a new entry rather than updating the old one. Old (larger) entries remain in the heap. The check if (d > dist[node]) continue discards them when they eventually pop.
TimeO((V+E) log V)
SpaceO(V + E)
ConstraintNon-negative weights only

🔴 Bellman-Ford

Bellman-Ford relaxes all edges V−1 times. Works with negative weights. A V-th relaxation pass that still improves a distance indicates a negative cycle. Slower than Dijkstra but more general.

☕ Java · Bellman-Ford 
int[] bellmanFord(int[][] edges, int n, int src) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE / 2);  // /2 to avoid overflow on relaxation
    dist[src] = 0;

    for (int i = 0; i < n - 1; i++) {        // V-1 rounds
        for (int[] e : edges) {                  // each edge {u, v, w}
            int u = e[0], v = e[1], w = e[2];
            if (dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
        }
    }
    // Optional: V-th pass to detect negative cycle
    for (int[] e : edges)
        if (dist[e[0]] + e[2] < dist[e[1]]) {
            // negative cycle reachable from src
        }
    return dist;
}
// Time: O(V * E)  Space: O(V)
💡
LC 787 “Cheapest Flights Within K Stops”: Use Bellman-Ford limited to K+1 rounds instead of V−1. Copy dist at the start of each round to avoid using the same round's updates within the same round (the “temp array” trick).
TimeO(V × E)
SpaceO(V)
HandlesNegative weights

🔳 Floyd-Warshall

Floyd-Warshall computes all-pairs shortest paths in O(V3) time. Works with negative weights (no negative cycles). Use only when V is small (≤ 300) and you need distances between every pair of nodes.

🎯 Floyd-Warshall Recognition Signal

"Shortest path between all pairs", "transitive closure", "can i reach j?" for every pair, or small V (≤ 300) with negative weights → Floyd-Warshall. Single source? Use Dijkstra or Bellman-Ford instead.

☕ Java · Floyd-Warshall All-Pairs Shortest Path 
// dist[i][j] = shortest distance from i to j
int INF = (int) 1e9;
int[][] dist = new int[n][n];
for (int[] row : dist) Arrays.fill(row, INF);
for (int i = 0; i < n; i++) dist[i][i] = 0;
for (int[] e : edges) dist[e[0]][e[1]] = e[2];  // {u, v, weight}

// Try every intermediate node k
for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            if (dist[i][k] < INF && dist[k][j] < INF)
                dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);

// Negative cycle: if dist[i][i] < 0 for any i, a negative cycle exists
// Time: O(V³)  Space: O(V²)
⚠
Overflow guard: Use INF = 1e9 (not Integer.MAX_VALUE) to avoid overflow when computing dist[i][k] + dist[k][j]. The guard if (dist[i][k] < INF) also protects you.
TimeO(V³)
SpaceO(V²)
HandlesNegative weights

🌳 Minimum Spanning Tree (MST)

An MST connects all V nodes using exactly V−1 edges with the minimum total weight. Two classic algorithms, both greedily build the MST.

Algorithm Strategy Data Structure Time Best When
Kruskal's Sort edges, add if no cycle Union-Find O(E log E) Sparse graphs
Prim's Grow MST from a node, always add cheapest reachable edge Min-Heap O(E log V) Dense graphs

Kruskal's Algorithm

☕ Java · Kruskal's MST (Union-Find) 
// edges: {weight, u, v}; sort by weight
int kruskal(int[][] edges, int n) {
    Arrays.sort(edges, (a, b) -> a[0] - b[0]);
    int[] parent = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;

    int cost = 0, edgesUsed = 0;
    for (int[] e : edges) {
        int w = e[0], u = e[1], v = e[2];
        int pu = find(parent, u), pv = find(parent, v);
        if (pu != pv) {  // different components — safe to add
            parent[pu] = pv;
            cost += w;
            if (++edgesUsed == n - 1) break;
        }
    }
    return edgesUsed == n - 1 ? cost : -1;  // -1 if not fully connected
}

int find(int[] parent, int x) {
    if (parent[x] != x) parent[x] = find(parent, parent[x]);  // path compression
    return parent[x];
}

Prim's Algorithm

☕ Java · Prim's MST (min-heap) 
int prim(List<List<int[]>> adj, int n) {
    boolean[] inMST = new boolean[n];
    // min-heap: {edge_weight, node}
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    pq.offer(new int[]{0, 0});  // start from node 0
    int cost = 0;

    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int w = cur[0], node = cur[1];
        if (inMST[node]) continue;
        inMST[node] = true;
        cost += w;
        for (int[] edge : adj.get(node))
            if (!inMST[edge[0]]) pq.offer(new int[]{edge[1], edge[0]});
    }
    return cost;
}
// Time: O(E log V)  Space: O(V + E)

🌸 Bipartite Graphs

A graph is bipartite if its nodes can be split into two groups such that every edge crosses between groups (no edge within a group). Equivalently: the graph is 2-colorable, equivalently: contains no odd-length cycle.

📌
Check: BFS 2-coloring. Assign alternating colors (0/1) during BFS. If you ever try to color a node the same color as its neighbour, the graph is not bipartite.
☕ Java · Bipartite Check (BFS coloring) 
boolean isBipartite(int[][] graph) {
    int n = graph.length;
    int[] color = new int[n];
    Arrays.fill(color, -1);

    for (int start = 0; start < n; start++) {
        if (color[start] != -1) continue;
        color[start] = 0;
        Queue<Integer> q = new LinkedList<>();
        q.offer(start);
        while (!q.isEmpty()) {
            int node = q.poll();
            for (int nei : graph[node]) {
                if (color[nei] == -1) {
                    color[nei] = 1 - color[node];
                    q.offer(nei);
                } else if (color[nei] == color[node]) {
                    return false;  // conflict
                }
            }
        }
    }
    return true;
}

🔣 Tarjan's Bridge Finding

A bridge is an edge whose removal disconnects the graph. Tarjan's algorithm finds all bridges in O(V + E) using two DFS arrays: disc[] (discovery time) and low[] (lowest discovery time reachable from the subtree without using the parent edge).

💡
Key insight: Edge (u, v) is a bridge when the subtree rooted at v cannot "look back" past u without this edge. Formally: low[v] > disc[u]. If low[v] ≤ disc[u], there is a back-edge in the subtree that bypasses the edge — removing it won't disconnect anything.
☕ Java · Tarjan's Bridge Finding (LC 1192) 
int[] disc, low;
int timer = 0;
List<List<Integer>> bridges = new ArrayList<>();

List<List<Integer>> criticalConnections(int n, List<List<Integer>> connections) {
    List<List<Integer>> adj = new ArrayList<>();
    for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
    for (List<Integer> e : connections) {
        adj.get(e.get(0)).add(e.get(1));
        adj.get(e.get(1)).add(e.get(0));
    }
    disc = new int[n]; low = new int[n];
    Arrays.fill(disc, -1);
    for (int i = 0; i < n; i++)
        if (disc[i] == -1) dfs(adj, i, -1);
    return bridges;
}

void dfs(List<List<Integer>> adj, int u, int parent) {
    disc[u] = low[u] = timer++;
    for (int v : adj.get(u)) {
        if (v == parent) continue;           // skip the edge we came from
        if (disc[v] == -1) {                  // tree edge — recurse
            dfs(adj, v, u);
            low[u] = Math.min(low[u], low[v]);
            if (low[v] > disc[u])             // bridge condition
                bridges.add(Arrays.asList(u, v));
        } else {                              // back edge — update low
            low[u] = Math.min(low[u], disc[v]);
        }
    }
}
// Time: O(V + E)  Space: O(V)
📌
Multigraph caveat: The parent-skip if (v == parent) fails when there are parallel edges (e.g., two edges between u and v). For multigraphs, track the parent edge index instead of the parent node.
TimeO(V + E)
SpaceO(V)

🎯 Pattern Decision Guide

Signal in Problem Pattern Key Tool
"Shortest path" — unweighted / unit weight BFS Queue + visited[]
"Shortest path" — weighted, non-negative Dijkstra Min-heap PQ
"Shortest path" — negative weights or K-stop limit Bellman-Ford Edge list, V−1 rounds
"Connected components", "number of islands" DFS flood fill visited[] + recursive DFS
"Course prerequisites", "task order", "dependency" Topological Sort Kahn's (in-degree) or DFS post-order
"Can you complete all courses" = detect cycle in directed graph Cycle Detection 3-color DFS or Kahn's (idx < n)
"Minimum cost to connect all nodes" MST Kruskal's (edges) or Prim's (heap)
"Two groups", "no conflict", "bipartite" BFS 2-coloring color[] alternating
"All-pairs shortest path" (small V) Floyd-Warshall 3-loop DP, O(V³)
✅
The 4-question framework for any graph problem:
1. What are the nodes? (cells, words, courses, …)
2. What are the edges? (adjacent cells, one-letter diff, prerequisite, …)
3. Directed or undirected? Weighted?
4. What am I computing? (shortest path, reachability, order, connectivity)

💪 In-Lecture Practice Problems

Problem 01 · LC 200
Number of Islands
Medium DFS Flood Fill Amazon · Google · Meta · Microsoft
▶
Problem

Given a 2D grid of '1' (land) and '0' (water), count the number of islands. An island is surrounded by water and connected horizontally/vertically.

Input: [["1","1","0"],["0","1","0"],["0","0","1"]] Output: 2
Intuition

DFS from each unvisited '1', marking all connected land cells as '0' (sinking the island). Each time you start a new DFS, increment the count. The grid itself acts as the visited array.

▶
Solution
☕ Java · LC 200 
int numIslands(char[][] grid) {
    int count = 0;
    for (int r = 0; r < grid.length; r++)
        for (int c = 0; c < grid[0].length; c++)
            if (grid[r][c] == '1') { dfs(grid, r, c); count++; }
    return count;
}
void dfs(char[][] g, int r, int c) {
    if (r < 0 || r >= g.length || c < 0 || c >= g[0].length || g[r][c] != '1') return;
    g[r][c] = '0';
    dfs(g,r+1,c); dfs(g,r-1,c); dfs(g,r,c+1); dfs(g,r,c-1);
}
TimeO(m×n)
SpaceO(m×n)
Problem 02 · LC 994
Rotting Oranges
Medium Multi-Source BFS Amazon · Google · Meta
▶
Problem

Grid contains 0 (empty), 1 (fresh orange), 2 (rotten). Each minute a rotten orange rots adjacent fresh oranges. Return the minimum minutes to rot all, or -1 if impossible.

Input: [[2,1,1],[1,1,0],[0,1,1]] Output: 4
Intuition

Seed all initially rotten oranges into the BFS queue at time 0. BFS propagates one level (= one minute) per round. After BFS completes, if any 1 remains in the grid, return -1.

▶
Solution
☕ Java · LC 994 
int orangesRotting(int[][] grid) {
    int rows = grid.length, cols = grid[0].length, fresh = 0, mins = 0;
    Queue<int[]> q = new LinkedList<>();
    for (int r=0; r<rows; r++) for (int c=0; c<cols; c++) {
        if (grid[r][c]==2) q.offer(new int[]{r,c});
        else if (grid[r][c]==1) fresh++;
    }
    int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
    while (!q.isEmpty() && fresh>0) {
        mins++;
        for (int sz=q.size(); sz>0; sz--) {
            int[] cur=q.poll();
            for (int[] d:dirs) {
                int nr=cur[0]+d[0], nc=cur[1]+d[1];
                if (nr>=0&&nr<rows&&nc>=0&&nc<cols&&grid[nr][nc]==1) {
                    grid[nr][nc]=2; fresh--; q.offer(new int[]{nr,nc});
                }
            }
        }
    }
    return fresh==0 ? mins : -1;
}
TimeO(m×n)
SpaceO(m×n)
Problem 03 · LC 207
Course Schedule
Medium Topological Sort / Cycle Detection Amazon · Google · Meta · Microsoft
▶
Problem

There are n courses (0 to n-1). prerequisites[i] = [a, b] means you must take b before a. Return true if it is possible to finish all courses.

Input: n=2, prerequisites=[[1,0]] Output: true Input: n=2, prerequisites=[[1,0],[0,1]] Output: false (cycle)
Intuition

Build a directed graph. A cycle means a course depends on itself (directly or indirectly) — impossible. Run Kahn's BFS: if the number of processed nodes equals n, no cycle.

▶
Solution
☕ Java · LC 207 
boolean canFinish(int n, int[][] pre) {
    List<List<Integer>> adj = new ArrayList<>();
    int[] indeg = new int[n];
    for (int i=0; i<n; i++) adj.add(new ArrayList<>());
    for (int[] p : pre) { adj.get(p[1]).add(p[0]); indeg[p[0]]++; }
    Queue<Integer> q = new LinkedList<>();
    for (int i=0; i<n; i++) if (indeg[i]==0) q.offer(i);
    int done = 0;
    while (!q.isEmpty()) {
        int node = q.poll(); done++;
        for (int nei : adj.get(node)) if (--indeg[nei]==0) q.offer(nei);
    }
    return done == n;
}
TimeO(V+E)
SpaceO(V+E)
Problem 04 · LC 743
Network Delay Time
Medium Dijkstra Google · Amazon · Uber
▶
Problem

Weighted directed graph with n nodes. Signal starts from node k. Return the time for all nodes to receive the signal, or -1 if any node is unreachable.

Input: times=[[2,1,1],[2,3,1],[3,4,1]], n=4, k=2 Output: 2
Intuition

Dijkstra from node k. Answer = max(dist[]). If any dist remains infinity, return -1. Classic single-source shortest path on a weighted graph.

▶
Solution
☕ Java · LC 743 
int networkDelayTime(int[][] times, int n, int k) {
    List<List<int[]>> adj = new ArrayList<>();
    for (int i=0; i<=n; i++) adj.add(new ArrayList<>());
    for (int[] t:times) adj.get(t[0]).add(new int[]{t[1],t[2]});
    int[] dist = new int[n+1];
    Arrays.fill(dist, Integer.MAX_VALUE); dist[k] = 0;
    PriorityQueue<int[]> pq = new PriorityQueue<>((a,b)->a[0]-b[0]);
    pq.offer(new int[]{0,k});
    while (!pq.isEmpty()) {
        int[] cur=pq.poll();
        if (cur[0]>dist[cur[1]]) continue;
        for (int[] e:adj.get(cur[1]))
            if (dist[cur[1]]+e[1]<dist[e[0]]) { dist[e[0]]=dist[cur[1]]+e[1]; pq.offer(new int[]{dist[e[0]],e[0]}); }
    }
    int ans = 0;
    for (int i=1; i<=n; i++) { if (dist[i]==Integer.MAX_VALUE) return -1; ans=Math.max(ans,dist[i]); }
    return ans;
}
TimeO((V+E) log V)
SpaceO(V+E)
Problem 05 · LC 1584
Min Cost to Connect All Points
Medium MST — Prim's Google · Amazon
▶
Problem

Given n points on a 2D plane, return the minimum cost to connect all points. Cost of connecting two points = Manhattan distance. All pairs are implicitly edges.

Input: [[0,0],[2,2],[3,10],[5,2],[7,0]] Output: 20
Intuition

MST on a complete graph where edge weight = Manhattan distance. Prim's is efficient here: maintain a min-heap of (cost, node). The inMST[] flag replaces the visited check. Greedily add the cheapest reachable edge.

▶
Solution
☕ Java · LC 1584 
int minCostConnectPoints(int[][] pts) {
    int n = pts.length;
    boolean[] inMST = new boolean[n];
    int[] minDist = new int[n];
    Arrays.fill(minDist, Integer.MAX_VALUE); minDist[0] = 0;
    PriorityQueue<int[]> pq = new PriorityQueue<>((a,b)->a[0]-b[0]);
    pq.offer(new int[]{0,0});
    int cost = 0;
    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        if (inMST[cur[1]]) continue;
        inMST[cur[1]] = true; cost += cur[0];
        for (int j=0; j<n; j++) {
            if (inMST[j]) continue;
            int d = Math.abs(pts[cur[1]][0]-pts[j][0]) + Math.abs(pts[cur[1]][1]-pts[j][1]);
            if (d < minDist[j]) { minDist[j]=d; pq.offer(new int[]{d,j}); }
        }
    }
    return cost;
}
TimeO(n² log n)
SpaceO(n)
