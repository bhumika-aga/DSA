Mastering the art of ordering data and finding elements in logarithmic time using partitioning, cycles, and binary search space.

1️⃣ Sorting Foundations: The \(O(N \log N)\) Barrier

Sorting is often the "hidden" first step in solving complex interval, mapping, or greedy problems. While \(O(N^2)\) sorts are easy to write, they fail at scale (\(N > 10^4\)). Interviewers expect you to default to \(O(N \log N)\) stability.

Algorithm Best Average Worst Memory Stable In-place
Bubble Sort \(O(N)\) \(O(N^2)\) \(O(N^2)\) \(O(1)\) Yes Yes
Selection Sort \(O(N^2)\) \(O(N^2)\) \(O(N^2)\) \(O(1)\) No Yes
Insertion Sort \(O(N)\) \(O(N^2)\) \(O(N^2)\) \(O(1)\) Yes Yes
Counting Sort \(O(N+K)\) \(O(N+K)\) \(O(N+K)\) \(O(K)\) Yes No
Merge Sort \(O(N \log N)\) \(O(N \log N)\) \(O(N \log N)\) \(O(N)\) Yes No
Quick Sort \(O(N \log N)\) \(O(N \log N)\) \(O(N^2)\) \(O(\log N)\) No Yes
Cyclic Sort \(O(N)\) \(O(N)\) \(O(N)\) \(O(1)\) No Yes
⚠️
Naive vs. Efficient: While \(O(N^2)\) sorts are rarely used for large datasets, they are foundational for learning swap logic and nested loops. Insertion Sort, specifically, is actually used inside Java's `Arrays.sort()` for very small arrays!

🫧 Bubble Sort: The Rising Bubbles

Compare adjacent elements. If they are in the wrong order, swap them. In each pass, the largest element "bubbles up" to its correct position at the end.

☕ Bubble Sort (Optimized)
public void bubbleSort(int[] arr) {
    boolean swapped;
    for (int i = 0; i < arr.length; i++) {
        swapped = false;
        for (int j = 1; j < arr.length - i; j++) {
            if (arr[j] < arr[j - 1]) {
                swap(arr, j, j - 1);
                swapped = true;
            }
        }
        if (!swapped) break; // Already sorted!
    }
}

🎯 Selection Sort: The Persistent Searcher

Find the minimum element in the unsorted part and swap it with the first element of that part. Repeat for the rest of the array.

☕ Selection Sort
public void selectionSort(int[] arr) {
    for (int i = 0; i < arr.length; i++) {
        int last = arr.length - i - 1;
        int maxIdx = getMaxIndex(arr, 0, last);
        swap(arr, maxIdx, last);
    }
}

🃏 Insertion Sort: The Card Player

Pick one element at a time and "insert" it into its correct position in the sorted part (like sorting playing cards in your hand).

☕ Insertion Sort
public void insertionSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++) {
        for (int j = i + 1; j > 0; j--) {
            if (arr[j] < arr[j - 1]) swap(arr, j, j - 1);
            else break;
        }
    }
}

🃏 Counting Sort: The Card Player

Pick one element at a time and "insert" it into its correct position in the sorted part (like sorting playing cards in your hand).

💡
Newbie Tip: Use Counting Sort when you have a million exam scores (0-100). Instead of comparing, just count how many 0s, 1s, etc. exist. It's like sorting fruit into baskets!
☕ Java Counting Sort
public void countingSort(int[] arr, int range) {
    int[] count = new int[range + 1];
    for (int x : arr) count[x]++;
    int idx = 0;
    for (int i = 0; i <= range; i++) {
        while (count[i]-- > 0) arr[idx++] = i;
    }
}
⚠️
"Manager" Analogy: A big task is hard. A manager splits the task between two seniors. The seniors split it between juniors. Once the juniors solve their tiny tasks, the manager combines the results. That is Divide & Conquer!

Merge Sort — The Stable Giant

Merge sort is the gold standard for stability and guaranteed performance. It always runs in \(O(N \log N)\) regardless of the input data.

☕ Merge Sort Full Implementation
void sort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2;
        sort(arr, l, m);
        sort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}

void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1, n2 = r - m;
    int L[] = new int[n1], R[] = new int[n2];
    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int j = 0; j < n2; j++) R[j] = arr[m + 1 + j];

    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}

Quick Sort — The In-Place Speedster

Quick sort picks a pivot and partitions the array around it. All elements smaller than the pivot go to the left, and larger ones to the right. It is widely used because it is in-place (\(O(\log N)\) stack space) and generally faster than Merge Sort due to cache locality.

📝 Quick Sort — Pseudo-code
// Partitioning
Pivot = Pick_Element(arr)
Move all < Pivot to Left
Move all > Pivot to Right
Return Pivot_Index

// Recursion
QuickSort(Left)
QuickSort(Right)
☕ Quick Sort (Lomuto Partition)
void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        int p = partition(arr, low, high);
        quickSort(arr, low, p - 1);
        quickSort(arr, p + 1, high);
    }
}

int partition(int[] arr, int low, int high) {
    int pivot = arr[high];
    int i = (low - 1);
    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr, i, j);
        }
    }
    swap(arr, i + 1, high);
    return i + 1;
}
⚠️
The Randomized Pivot: If you always pick the first or last element as the pivot, QuickSort degrades to \(O(N^2)\) for already sorted arrays. In competitive programming, always use `swap(arr, low + random.nextInt(high - low), high)` before partitioning!
☕ QuickSelect — Kth Largest in O(N)
public int quickSelect(int[] nums, int low, int high, int k) {
    int p = partition(nums, low, high);
    if (p == k) return nums[p];
    if (p > k) return quickSelect(nums, low, p - 1, k);
    return quickSelect(nums, p + 1, high, k);
}
INSIGHT
QuickSelect: Finding Kth Smallest in O(N)

By only recurring into the partition that contains our target index \(K\), we can find the Kth smallest element in \(O(N)\) average time instead of \(O(N \log N)\).

📉 Sorting in Linear Time

When we know the range of the input (e.g., scores from 0-100), we can skip comparisons entirely. Comparison-based sorts (Merge/Quick) are bounded by \(O(N \log N)\), but \(O(N)\) is possible here.

☕ Counting Sort (Standard)
void countingSort(int[] arr) {
    int max = Arrays.stream(arr).max().getAsInt();
    int min = Arrays.stream(arr).min().getAsInt();
    int range = max - min + 1;
    int[] count = new int[range];
    for (int x : arr) count[x - min]++;
    int k = 0;
    for (int i = 0; i < range; i++) {
        while (count[i]-- > 0) arr[k++] = i + min;
    }
}
🏗️
Radix & Bucket Sort: For large ranges, we sort digit by digit (Radix) or distribute elements into uniform bins (Bucket). Essential for sorting floating-point numbers in nearly-linear time.

3️⃣ 🔄 Cyclic Sort: The \(O(N)\) Interview Secret

Whenever an array contains numbers in a range (e.g., \(1\) to \(N\)), use Cyclic Sort to place each number at its matching index. This pattern solves 90% of "find missing/duplicate" problems.

☕ Cyclic Sort Problem 01 · Find the Missing Number
public int missingNumber(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        if (nums[i] < nums.length && nums[i] != nums[nums[i]]) {
            swap(nums, i, nums[i]);
        } else i++;
    }
    for (i = 0; i < nums.length; i++) {
        if (nums[i] != i) return i;
    }
    return nums.length;
}
☕ Cyclic Sort Problem 02 · Find All Duplicates
public List<Integer> findDuplicates(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int correct = nums[i] - 1;
        if (nums[i] != nums[correct]) swap(nums, i, correct);
        else i++;
    }
    List<Integer> res = new ArrayList<>();
    for (i = 0; i < nums.length; i++) {
        if (nums[i] != i + 1) res.add(nums[i]);
    }
    return res;
}
📝 Cyclic Sort — Pseudo-code
// For numbers in range [1, N]
While i < N:
   Correct_Index = arr[i] - 1
   If arr[i] != arr[Correct_Index]:
      Swap(arr[i], arr[Correct_Index])
   Else:
      i++
💡
Cyclic Sort vs Floyd's: While Cyclic Sort is great for finding missing/duplicate members in \(1 \dots N\), Floyd’s Cycle Finding (Hare & Tortoise) is superior when you aren't allowed to modify the array.

4️⃣ 📊 Searching Roadmap

Before diving into implementations, understand the landscape. Your choice of algorithm depends entirely on the data's structure, distribution, and access speed.

Algorithm Complexity Pre-condition Best Use Case
Linear Search \(O(N)\) None Small / Unsorted Data
Jump Search \(O(\sqrt{N})\) Sorted Costly backward traversal
Binary Search \(O(\log N)\) Sorted / Monotonic Standard Large Datasets
Interpolation Search \(O(\log \log N)\) Sorted + Uniform Telephone Directories / Uniform Numerical Data

5️⃣ 🔎 The Searching Paradigm (Non-Binary)

Searching is the most fundamental operation in computation. For unsorted data or specific distributions, Binary Search is either impossible or sub-optimal.

5.1 Linear Search — The Brute Force Baseline

If the data is unsorted, you must check every element. It is \(O(N)\) because the target could be anywhere.

☕ Standard Linear Search
public int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) return i;
    }
    return -1;
}

5.2 Jump Search — The Square Root Optimization

Instead of checking one-by-one, jump in blocks of size \(m = \sqrt{N}\). If `arr[m]` is greater than target, you know the target lies in the current block. Perform a linear search within that block.

💡
Why Jump? In systems where jumping back is expensive (e.g., tape drives or specific file streams), Jump Search is preferred over Binary Search as it only requires one jump back.

5.3 Interpolation Search — The Intelligent Guess

If you look for "Zebra" in a dictionary, you don't start in the middle; you go to the end. Interpolation search calculates a probe position based on the values of the boundaries.

Formula: \(pos = lo + \frac{(target - arr[lo]) \cdot (hi - lo)}{(arr[hi] - arr[lo])}\)

⚠
Warning: Interpolation search is only \(O(\log \log N)\) if the data is uniformly distributed. For skewed data, it can degrade to \(O(N)\).

6️⃣ 🔍 Binary Search Masterclass

Binary Search isn't just a code block; it's a law of nature. By halving the search space, we find a needle in 4 billion items in just 32 steps (\(O(\log N)\)).

6.1 Foundations & Overflow Trap

📝 Binary Search — Pseudo-code
// The Immortal Pattern
While Low <= High:
   Mid = Low + (High - Low) / 2 // Avoid Overflow!
   If arr[Mid] == Target: Return Mid
   Else if arr[Mid] < Target: Low = Mid + 1
   Else: High = Mid - 1
Return -1
🌋
The Integer Overflow Trap:
Never use `mid = (low + high) / 2`. If `low` and `high` are \(10^9\), their sum will exceed the integer max (\(2^{31}-1\)). Always use:
mid = low + (high - low) / 2

6.2 Master Templates (Standard & Boundary)

Standard search finds any instance. Boundary search finds the edge instance.

Scenario While Condition Success Logic Movement
Exact Match `low <= high` `mid == target` Standard return
First Occurrence `low < high` Store index `high = mid - 1`
Last Occurrence `low < high` Store index `low = mid + 1`

6.3 Modified Variations (Rotated, Peak, 2D)

BS works as long as there is a monotonic property (increasing/decreasing) or a predictable gradient.

🔄
Rotated Array: Even when rotated, one half (\(mid \rightarrow hi\) or \(lo \rightarrow mid\)) must be sorted. Decision logic: Check if target lies within the boundaries of the sorted half.
🌋
Find Peak Element (Gradients): Compare `mid` with `mid+1`. If `arr[mid] < arr[mid+1]`, the peak is to the right. You are using the slope to find the summit.

7️⃣ Binary Search on Answer Space

The most powerful form of BS. Instead of searching in an array, we search for a specific value in a range that satisfies a boolean predicate.

📝 BS on Answer — Expert Template
Low = Min_Possible_Answer, High = Max_Possible_Answer
Result = -1

While Low <= High:
   Mid = Low + (High - Low) / 2
   If isPossible(Mid):
      Result = Mid
      High = Mid - 1 // Try to find a smaller valid answer (Minimize)
   Else:
      Low = Mid + 1 // Answer must be larger
🎯
Interview Pattern: If a problem asks to "Minimize the Maximum" or "Maximize the Minimum", it is almost always Binary Search on Answer. (e.g., Aggressive Cows, Book Allocation, Split Array Largest Sum).

6.4 Advanced BS (Exponential, Ternary)

♾️
Exponential Search: For infinite streams or very large arrays where you don't know the end. Double your range (\(1, 2, 4, 8...\)) until you overshot the target, then BS the range.
📐
Ternary Search: Splits space into three parts (\(m1, m2\)). Best for finding extreme points (peaks) in unimodal functions. \(O(\log_3 N)\).

6.5 Binary Search on Answer Space

The "Master Tier". You aren't searching for a needle in a haystack; you are searching for the optimal value in a monotonic range of possibilities (e.g., minimum capacity, maximum distance).

☕ BS on Answer Space · Example Pattern
while (low <= high) {
    int mid = low + (high - low) / 2;
    if (isPossible(mid)) {
        ans = mid; high = mid - 1; // Try for more "optimal" (smaller)
    } else low = mid + 1;
}
🍌
Koko Eating Bananas Strategy:
1. Range: \([1, \max(piles)]\).
2. Predicate: `canEatInTime(speed)`. If she can eat at speed \(S\), she can certainly eat at \(S+1\).
☕ Binary Search · Koko Eating Bananas
public int minEatingSpeed(int[] piles, int h) {
    int low = 1, high = 1000000000;
    int ans = high;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (canFinish(piles, h, mid)) {
            ans = mid; high = mid - 1;
        } else low = mid + 1;
    }
    return ans;
}

7️⃣ ⚖️ Custom Comparators

In real-world DSA, we rarely sort simple integers. We sort objects (Students by Rank, Orders by Priority, Intervals by Start Time).

7.1 Comparable vs Comparator

Feature Comparable Comparator
Location Defined inside the class External logic
Control "Natural" sorting order Multiple sorting strategies
Method `compareTo(T obj)` `compare(T o1, T o2)`
💡
The Subtract Trick: For integers `(a, b) -> a - b`.
If result is negative, `a` comes first. If positive, `b` comes first. Use `(a, b) -> b - a` for descending order.
☕ Modern Comparator Syntax (Java 8+)
// 1. Sorting a 2D Array by the first column
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);

// 2. Multi-Level Sorting: By Age, then by Name
Collections.sort(students, Comparator
    .comparingInt(Student::getAge)
    .thenComparing(Student::getName));

public int search(int[] nums, int target) {
    int lo = 0, hi = nums.length - 1;

        int mid = lo + (hi - lo) / 2;

Problem 02 · Boundary Search
Find First and Last Position

Problem

Given a sorted array, find the starting and ending index of a given target value.

▶Full solution
☕ Java
public int[] searchRange(int[] nums, int target) {
    int first = find(nums, target, true);
    int last = find(nums, target, false);
    return new int[]{first, last};
}
private int find(int[] nums, int target, boolean isFirst) {
    int res = -1, lo = 0, hi = nums.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) {
            res = mid;
            if (isFirst) hi = mid - 1;
            else lo = mid + 1;
        } else if (nums[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return res;
}
TimeO(log N)
SpaceO(1)
Problem 03 · Modified Binary Search
Search in Rotated Sorted Array
🔗 LC 33 - Rotated Search
Medium GoogleMicrosoft Partially Sorted
›
Problem

Search target in a sorted array that has been rotated at some unknown pivot.

▶Full solution
☕ Java
public int search(int[] nums, int target) {
    int lo = 0, hi = nums.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return mid;
        if (nums[lo] <= nums[mid]) { // Left side is sorted
            if (target >= nums[lo] && target < nums[mid]) hi = mid - 1;
            else lo = mid + 1;
        } else { // Right side is sorted
            if (target > nums[mid] && target <= nums[hi]) lo = mid + 1;
            else hi = mid - 1;
        }
    }
    return -1;
}
TimeO(log N)
SpaceO(1)

Problem 05 · Cycle Hashing
Find All Numbers Disappeared
🔗 LC 448 - Missing Numbers
Easy GoogleMicrosoft Cyclic Sort

Problem

Find all elements in range [1, n] that do not appear in the array.

▶Full solution
☕ Java
public List<Integer> findDisappearedNumbers(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int correctIndex = nums[i] - 1;
        if (nums[i] != nums[correctIndex]) {
            swap(nums, i, correctIndex);
        } else i++;
    }
    List<Integer> res = new ArrayList<>();
    for (i = 0; i < nums.length; i++) {
        if (nums[i] != i + 1) res.add(i + 1);
    }
    return res;
}

SpaceO(1)
Problem 06 · Standard Merge
Merge Sorted Array (In-Place)

Problem

Merge two sorted arrays into one sorted array in-place (nums1 has extra space).

▶Full solution
☕ Java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    int i = m - 1, j = n - 1, k = m + n - 1;
    while (j >= 0) {
        if (i >= 0 && nums1[i] > nums2[j]) nums1[k--] = nums1[i--];
        else nums1[k--] = nums2[j--];
    }
}
TimeO(M+N)
SpaceO(1)
Problem 07 · Stable Sorting
Sort List (Merge Sort on Nodes)
🔗 LC 148 - Merge Sort LL
Medium AdobeGoldman D&C
›
Problem

Sort a linked list in \(O(n \log n)\) time using constant space complexity.

▶Full solution
☕ Java
public ListNode sortList(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode prev = null, slow = head, fast = head;
    while (fast != null && fast.next != null) {
        prev = slow; slow = slow.next; fast = fast.next.next;
    }
    prev.next = null; // Cut list
    ListNode l1 = sortList(head);
    ListNode l2 = sortList(slow);
    return merge(l1, l2);
}
TimeO(N log N)
SpaceO(log N) stack

Problem 09 · Average O(N) Selection

Find the kth largest element in an unsorted array. \(O(N)\) average time required.

public int findKthLargest(int[] nums, int k) {
    k = nums.length - k; // Find target index
    int lo = 0, hi = nums.length - 1;
    while (lo < hi) {
        int j = partition(nums, lo, hi);
        if (j < k) lo = j + 1;
        else if (j > k) hi = j - 1;
        else break;
    }
    return nums[k];
}

▶Full solution

public int firstMissingPositive(int[] nums) {
    int i = 0;
    while (i < nums.length) {
        int correct = nums[i] - 1;
        if (nums[i] > 0 && nums[i] <= nums.length && nums[i] != nums[correct]) {
            swap(nums, i, correct);
        } else i++;
    }
    for (i = 0; i < nums.length; i++) {
        if (nums[i] != i + 1) return i + 1;
    }
    return nums.length + 1;
}
