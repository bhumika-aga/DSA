Mastering contiguous blocks of memory with mathematical tricks, frequencies, and in-place manipulation.

Why Arrays matter at scale: Arrays are the single most cache-friendly data structure in computer science. Modern CPUs pull contiguous chunks of memory into the L1 cache. A linear scan of an array is orders of magnitude faster than traversing a Linked List, even if both are O(N).

1️⃣ Prefix Sums (1D and 2D)

Imagine your bank account. If you want to know exactly how much you spent between Wednesday and Friday, you don't need to manually add up Wednesday + Thursday + Friday's receipts. You can simply look at your total balance on Friday and subtract your total balance from Tuesday!

This is the essence of Prefix Sums. They allow you to answer multiple Range Sum Queries in O(1) time after a single O(N) precomputation step.

The 1D Memory Logic

We build a new array P where P[i] stores the sum of all elements from index 0 up to i.

// Original Array (Daily Expenses): arr = [ 3, 1, 4, 1, 5 ] // Prefix Sum Array (Cumulative Spending): // P[0] = 3 // P[1] = P[0] + 1 = 4 // P[2] = P[1] + 4 = 8 P = [ 3, 4, 8, 9, 14 ]
☕ Java
int[] P = new int[arr.length];
P[0] = arr[0];
for (int i = 1; i < arr.length; i++) {
    P[i] = P[i - 1] + arr[i];
}

To find the sum of range [L, R]:

  • If L == 0: Return P[R] (You want the sum from the very beginning).
  • If L > 0: Return P[R] - P[L - 1] (You drop all the "baggage" that came before L).

The 2D Matrix Logic

What if you have a 2D matrix (like a grid of numbers) and you want the sum of a specific rectangle? Doing nested loops takes an incredibly slow O(N²M²). Instead, we use the Inclusion-Exclusion Principle (think Venn Diagrams).

Operation Formula Statement
Precompute `P[i][j]` arr[i][j] + P[i-1][j] + P[i][j-1] - P[i-1][j-1]
Add the cell itself + the rectangle ABOVE + the rectangle LEFT. Since the top-left corner was added twice, subtract it once!
Query `(r1,c1)` to `(r2,c2)` P[r2][c2] - P[r1-1][c2] - P[r2][c1-1] + P[r1-1][c1-1]
Take the giant rectangle ending at (r2, c2). Cut off the top part. Cut off the left part. Since you cut out the top-left corner TWICE, add it back once!

2️⃣ Kadane's Algorithm — Maximum Subarray

If you're looking for the largest possible sum from a contiguous chunk of an array, checking every chunk takes O(n³). Sliding over them takes O(n²). Kadane's algorithm discovers the answer in a blazingly fast O(N) single pass!

The "Reset" Baggage Intuition

Kadane’s works on a simple philosophy: If my past is dragging me down (negative sum), I must drop the baggage and start fresh!

At every step, you must make a greedy choice: Do I append the current number to the existing subarray chain, OR do I cut ties and start a brand-new subarray at this exact number?

Let's dry run the array: [-2, 1, -3, 4] [ -2 ] -> currentMax = -2. (It's terrible, but it's the only subarray so far.) [ 1 ] -> Do I append 1 to -2 (gives -1)? Or start fresh at 1? (Clearly, 1 > -1. Start fresh!) currentMax = 1 [ -3 ] -> Do I append -3 to 1 (gives -2)? Or start fresh at -3? (-2 is technically better than -3. Keep the chain!) currentMax = -2 [ 4 ] -> Do I append 4 to -2 (gives 2)? Or start fresh at 4? (4 > 2. Drop the baggage!) currentMax = 4 Global Max recorded during this trip: 4
☕ Java · Maximum Subarray
public int maxSubArray(int[] nums) {
    int currentMax = nums[0];
    int globalMax = nums[0];
    
    // Start loop at index 1!
    for (int i = 1; i < nums.length; i++) {
        // The Core Philosophy: Do I accept the baggage or ditch it?
        currentMax = Math.max(nums[i], currentMax + nums[i]);
        globalMax = Math.max(globalMax, currentMax);
    }
    return globalMax;
}
🌟
Extensions: You can apply Kadane's logic to find the Max Product Subarray (track both `minProduct` and `maxProduct` because a negative * negative = huge positive!) or the Best Time to Buy & Sell Stock (track the `minPrice` backwards).

3️⃣ Dutch National Flag (Three-Way Partition)

What if you are given an array of 0s, 1s, and 2s and told to sort them in a single pass in O(1) space? Invented by Edsger Dijkstra, the DNF approach maintains 4 strictly defined zones using 3 pointers: low, mid, and high.

Analyzing 3 Zones

The array starts entirely in the "Unknown" zone. The goal is to shrink the Unknown zone until `mid` passes `high`.

  • 0 to low - 1: The confirmed 0s zone.
  • low to mid - 1: The confirmed 1s zone.
  • mid to high: The UNEXPLORED zone.
  • high + 1 to N - 1: The confirmed 2s zone.
Before sorting: ALL elements are between mid and high (Unknown) [ 2, 0, 2, 1, 1, 0 ] ↑ ↑ mid(low) high If nums[mid] == 0: Swap(low, mid), increment BOTH. (1s shift right) If nums[mid] == 1: Increment mid. (It's already in the middle) If nums[mid] == 2: Swap(mid, high), decrement high. (DO NOT increment mid!)
⚠️
Why don't we increment `mid` when swapping a `2` with `high`?
Because when you pull an element from the `high` index into the `mid` index, that element came from the Unexplored Zone. You don't know if the number you just pulled is a 0, 1, or 2! It must be processed again by the loop.
☕ Java · Sort Colors
public void sortColors(int[] nums) {
    // Initialize the strict bounds
    int low = 0, mid = 0, high = nums.length - 1;
    
    while (mid <= high) {
        if (nums[mid] == 0) {
            swap(nums, low, mid);
            low++; mid++;
        } else if (nums[mid] == 1) {
            mid++;
        } else { // nums[mid] == 2
            swap(nums, mid, high);
            high--; // Warning! Don't increment mid!
        }
    }
}

4️⃣ Frequency Arrays vs HashMaps

When solving an Anagram substring string count problem, the instinct is to use `HashMap`. But what if you are strictly mapping lowercase english letters?

HashMaps are painfully slow. They require creating an Object for every Key/Value pair, suffer from hashing collisions, and cause Memory Fragmentation. If the character set is just `'a' - 'z'`, a flat int[26] array is the absolute fastest way to count in O(1) time!

The Character Mapping Trick

In ASCII, `'a'` is 97, `'b'` is 98.
Therefore, 'c' - 'a' = 99 - 97 = 2.

Mapping 'a' to 'z' strictly into Indexes 0 through 25 char: 'a' 'b' 'c' ... 'z' idx: [ 0 1 2 ... 25 ]
☕ Java · Valid Anagram O(N)
public boolean isAnagram(String s, String t) {
    // If lengths don't match, they trivially aren't anagrams!
    if (s.length() != t.length()) return false;
    
    // Only 26 english letters!
    int[] count = new int[26];
    
    for (int i = 0; i < s.length(); i++) {
        count[s.charAt(i) - 'a']++;  // 'c' maps to count[2]
        count[t.charAt(i) - 'a']--;  // Balance exactly on the opposite side
    }
    
    // If any count isn't exactly 0, they had different letter distributions.
    for (int val : count) {
        if (val != 0) return false;
    }
    return true;
}

5️⃣ The Reversal Algorithm (Rotate Array)

How do we rotate an array by k steps using O(1) auxiliary space? Moving elements one by one takes O(n*k). The math trick is three simple reversals.

☕ Java · Rotate by K
public void rotate(int[] nums, int k) {
    k = k % nums.length;
    // Step 1: reverse the entire array
    reverse(nums, 0, nums.length - 1);
    // Step 2: reverse the first k elements
    reverse(nums, 0, k - 1);
    // Step 3: reverse the remaining rest
    reverse(nums, k, nums.length - 1);
}

6️⃣ String Immutability & StringBuilder

In Java, Python, and C#, Strings are immutable. Once created, they cannot be changed. This means `s += "a"` creates a brand-new string in memory and copies all existing characters over!

⚠️
The O(N²) Trap: If you append N characters in a loop using +=, you perform $1 + 2 + 3 + ... + N = O(N^2)$ array copies. You will get TLE (Time Limit Exceeded) on LeetCode.

The Fix: Always use a dynamic resizing array buffer for loops.

☕ Java · StringBuilder
// O(N) Time - Amortized constant time appends
StringBuilder sb = new StringBuilder();
for (char c : chars) {
    sb.append(c);
}
return sb.toString();

7️⃣ Rabin-Karp & Rolling Hash Intro

If you search for a substring of length M in a string of length N, the naive match is O(N × M). Rabin-Karp converts the substring to a numerical hash to do this in O(N) overall time.

The Mechanics: A "Rolling Hash" allows you to slide a window by conceptually subtracting the old character dropping out, and adding the new character shifting in, doing this math in strictly O(1) time per shift instead of recomputing the M length hash from scratch.

🧠
Rolling formula: hash = (hash - (oldChar * base^(M-1))) * base + newChar.
You will apply this heavily in finding duplicate substrings and pattern matching.

8️⃣ 2D Boundary Traversals (Spiral & Zigzag)

One of the most intimidating array problems in technical interviews is navigating a 2D matrix in non-linear order, like spiraling inwards or zigzagging across diagonals. The trick is to stop thinking of loops going from 0 to N, and instead think of 4 dynamic, shrinking walls.

The Shrinking Wall Analogy

Picture a physical room where you are painting the floor. You paint the Top wall, then the Right wall, then the Bottom wall, then the Left wall. After doing a full perimeter lap, the "unpainted" zone has shrunk by 1 layer on all sides! You keep painting the new boundary until your walls crash into each other.

Matrix: [ 1, 2, 3 ] T = 0 (Top Wall Row) [ 4, 5, 6 ] B = 2 (Bottom Wall Row) [ 7, 8, 9 ] ↑ ↑ L=0 R=2 (Left/Right Cols) Step 1 (Paint Top): [1, 2, 3] -> Move T down. Step 2 (Paint Right): [6, 9] -> Move R left. Step 3 (Paint Bot): [8, 7] -> Move B up. Step 4 (Paint Left): [4] -> Move L right. New Walls: T=1, B=1, L=1, R=1. (Only '5' is unpainted, loop repeats and gets it!)
☕ Java · Spiral Matrix
public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> res = new ArrayList<>();
    int top = 0, bot = matrix.length - 1;
    int left = 0, right = matrix[0].length - 1;
    
    while (top <= bot && left <= right) {
        // Paint Top Wall
        for (int i = left; i <= right; i++) res.add(matrix[top][i]);
        top++;
        
        // Paint Right Wall
        for (int i = top; i <= bot; i++) res.add(matrix[i][right]);
        right--;
        
        // Prevent duplicate painting if walls crossed!
        if (top <= bot) {
            // Paint Bottom Wall
            for (int i = right; i >= left; i--) res.add(matrix[bot][i]);
            bot--;
        }
        if (left <= right) {
            // Paint Left Wall
            for (int i = bot; i >= top; i--) res.add(matrix[i][left]);
            left++;
        }
    }
    return res;
}

Problem

You are given an n x n 2D matrix representing an image, rotate the image by 90 degrees (clockwise) in-place.

▶Full solution
☕ Java
public void rotate(int[][] matrix) {
    int n = matrix.length;
    // 1. Transpose: Swap matrix[i][j] with matrix[j][i]
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            int tmp = matrix[i][j];
            matrix[i][j] = matrix[j][i];
            matrix[j][i] = tmp;
        }
    }
    // 2. Flip horizontally: Reverse each row
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n / 2; j++) {
            int tmp = matrix[i][j];
            matrix[i][j] = matrix[i][n - 1 - j];
            matrix[i][n - 1 - j] = tmp;
        }
    }
}
TimeO(N²)
SpaceO(1)

public int maxSubArray(int[] nums) {
    int maxSoFar = nums[0], currentMax = nums[0];
    for (int i = 1; i < nums.length; i++) {
        // Decision: Extend existing subarray or start fresh?

public void sortColors(int[] nums) {
    int low = 0, mid = 0, high = nums.length - 1;
    while (mid <= high) {
        if (nums[mid] == 0) {
            swap(nums, low++, mid++);
        } else if (nums[mid] == 1) {
            mid++;
        } else {
            swap(nums, mid, high--);
        }
    }
}

Problem 04 · Prefix Sum + HashMap
Subarray Sum Equals K
🔗 LC 560 - Subarray Sum K
Medium GoogleMeta Prefix Diff
›
Problem

Find the total number of continuous subarrays whose sum equals k.

▶Full solution
☕ Java
public int subarraySum(int[] nums, int k) {
    int count = 0, sum = 0;
    HashMap<Integer, Integer> map = new HashMap<>();
    map.put(0, 1);
    for (int n : nums) {
        sum += n;
        if (map.containsKey(sum - k)) count += map.get(sum - k);
        map.put(sum, map.getOrDefault(sum, 0) + 1);
    }
    return count;
}
TimeO(N)
SpaceO(N)

public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;
    int[] counts = new int[26];
    for (char c : s.toCharArray()) counts[c - 'a']++;
    for (char c : t.toCharArray()) {
        if (--counts[c - 'a'] < 0) return false;
    }
    return true;
}
TimeO(N)
SpaceO(1)
Problem 06 · Prefix/Suffix Product
Product of Array Except Self
🔗 LC 238 - Product Except Self
Medium AmazonApple Two Pass
›
Problem

Calculate the product of all elements except the current one without using division.

▶Full solution
☕ Java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] res = new int[n];
    res[0] = 1;
    for (int i = 1; i < n; i++) res[i] = res[i - 1] * nums[i - 1];
    int right = 1;
    for (int i = n - 1; i >= 0; i--) {
        res[i] *= right;
        right *= nums[i];
    }
    return res;
}
TimeO(N)
SpaceO(1)

public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> res = new ArrayList<>();
    int top = 0, bottom = matrix.length - 1;
    int left = 0, right = matrix[0].length - 1;
    while (top <= bottom && left <= right) {
        for (int j = left; j <= right; j++) res.add(matrix[top][j]);
        top++;
        for (int i = top; i <= bottom; i++) res.add(matrix[i][right]);
        right--;
        if (top <= bottom)
            for (int j = right; j >= left; j--) res.add(matrix[bottom][j]);
        bottom--;
        if (left <= right)
            for (int i = bottom; i >= top; i--) res.add(matrix[i][left]);
        left++;
    }
    return res;
}
TimeO(M*N)
SpaceO(1)
Problem 08 · Sliding Window
Longest Substring Without Repeating
🔗 LC 3 - Longest Non-Repeating
Medium AmazonGoogle Two Pointers
›
Problem

Find the length of the longest substring without repeating characters.

▶Full solution
☕ Java
public int lengthOfLongestSubstring(String s) {
    int maxLen = 0, left = 0;
    HashSet<Character> set = new HashSet<>();
    for (int right = 0; right < s.length(); right++) {
        while (set.contains(s.charAt(right))) {
            set.remove(s.charAt(left++));
        }
        set.add(s.charAt(right));
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
TimeO(N)
SpaceO(min(M, N))

public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] ca = s.toCharArray();
        Arrays.sort(ca);
        String key = String.valueOf(ca);
        if (!map.containsKey(key)) map.put(key, new ArrayList<>());
        map.get(key).add(s);
    }
    return new ArrayList<>(map.values());
}
TimeO(N * K log K)
SpaceO(N * K)

Problem

Compress an array of characters in-place using constant extra space.

▶Full solution
☕ Java
public int compress(char[] chars) {
    int index = 0, i = 0;
    while (i < chars.length) {
        int j = i;
        while (j < chars.length && chars[j] == chars[i]) j++;
        chars[index++] = chars[i];
        if (j - i > 1) {
            for (char c : String.valueOf(j - i).toCharArray()) chars[index++] = c;
        }
        i = j;
    }
    return index;
}
TimeO(N)
SpaceO(1)
