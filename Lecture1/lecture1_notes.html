🧱 Data Types

1.1 — Primitive Types (8 total)

Stored directly on the stack. Fixed size, no null, ultra-fast access. These are the atoms of Java.

Type Bits Range Default DSA Usage
byte 8 −128 → 127 0 raw byte buffers, rarely in DSA
short 16 −32,768 → 32,767 0 rarely used
int ⭐ 32 −2.1B → 2.1B 0 default integer — use this always
long ⭐ 64 ±9.2 × 10¹⁸ 0L large products, timestamps, overflow-prone problems
float 32 ~6-7 sig. digits 0.0f avoid — precision issues
double ⭐ 64 ~15-16 sig. digits 0.0 default decimal
char ⭐ 16 0 → 65,535 (Unicode) \u0000 frequency arrays, char arithmetic
boolean 1 bit true / false false flags, visited arrays, conditions
💡
DSA Rule of Thumb for type choice:
• Default to int for all integers.
• Switch to long when result can exceed ~2.1 billion (e.g. n × m with n,m ≈ 10⁵).
• char - 'a' gives you an index 0–25 — this is how you build O(1) frequency arrays for lowercase letters.
• double for geometry, probabilities, averages.

1.2 — Reference (Non-Primitive) Types

Variables hold a memory address (reference) pointing to an object on the heap. Can be null. Includes all classes, arrays, interfaces.

☕ Java
// Primitive — value lives IN the variable (stack)
int x = 42;           // x IS 42

// Reference — variable holds ADDRESS pointing to heap
String s = "hello";  // s holds address of "hello" object on heap
int[] arr = {1,2,3};  // arr holds address of the array on heap

// Consequence: == checks identity (same address), not equality!
String a = new String("hi");
String b = new String("hi");
System.out.println(a == b);       // false  (different addresses!)
System.out.println(a.equals(b));  // true   (same content) ← USE THIS

1.3 — Type Casting (Interview Gotchas)

☕ Java · Casting Traps
// Widening (implicit, safe) — smaller → larger
int i = 100;
long l = i;          // auto-widens: no data loss
double d = i;        // auto-widens to 100.0

// Narrowing (explicit, lossy) — larger → smaller
double pi = 3.99;
int truncated = (int) pi;   // → 3  (TRUNCATES, does NOT round!)

// ⚠️ GOTCHA 1: integer division truncates
double wrong   = 5 / 2;      // → 2.0  (int division first, THEN widened)
double correct = 5.0 / 2;    // → 2.5  (double division)
double correct2 = (double) 5 / 2; // → 2.5  (cast first)

// ⚠️ GOTCHA 2: int overflow — use long!
int n = 100000;
int overflow = n * n;         // → negative! (wraps around 2.1B limit)
long safe = (long) n * n;    // → 10000000000L ✓

// ⭐ char arithmetic — essential for DSA
int idx = 'e' - 'a';            // → 4  (freq array index)
char c = (char)('a' + 3);       // → 'd'
boolean isLower = (ch >= 'a' && ch <= 'z'); // char range check
char upper = (char)(ch - 32);   // lowercase → uppercase
⚠️
Autoboxing / Unboxing Trap — Common Interview Gotcha!
Java caches Integer values from −128 to 127 (the Integer cache). Outside this range each auto-boxing creates a new heap object.

Integer a = 127, b = 127; a == b → true (same cached object)
Integer c = 128, d = 128; c == d → false (two different heap objects!)

Rule: ALWAYS use .equals() to compare wrapper types — never ==.
Additional trap: Unboxing a null Integer to int silently throws NullPointerException!

🔁 Control Flow

2.1 — All Loop Types

🎯
Pattern Recognition: Which loop to use?
• for: When you know the exact iteration count upfront.
• while: When iteration count depends on a condition (unknown upfront). Classic for binary search, two-pointer, digit extraction.
• do-while: When the body must execute at least once. Classic for menu loops, digit extraction where n=0 is possible.
• for-each: Cleanest for read-only traversal of collections/arrays when you don't need the index.
☕ Java · All Loop Types + DSA Usage
// ── FOR loop — use when count is known ───────────────────────
for (int i = 0; i < n; i++)          // forward: i goes 0..n-1
for (int i = n-1; i >= 0; i--)        // reverse: suffix processing
for (int i = 0; i < n; i += 2)        // step 2: even indices
for (int i = 0; i < n-1; i++)         // n-1 pairs: adj comparison

// ── WHILE loop — use when count is unknown ───────────────────
while (lo <= hi) { ... }              // binary search template
while (n > 0) { digit = n%10; n/=10;} // digit extraction
while (n > 1) { n /= 2; count++; }   // count divisions → O(log n)
while (fast != null && fast.next != null) // linked list traversal

// ── DO-WHILE — body runs at least once ───────────────────────
do {
    digit = n % 10;
    reversed = reversed * 10 + digit;
    n /= 10;
} while (n != 0);                    // handles n=0 correctly

// ── FOR-EACH — read-only traversal ───────────────────────────
for (int x : arr) sum += x;           // array
for (String s : list) process(s);    // collection
for (Map.Entry<K,V> e : map.entrySet()) // map iteration

// ── NESTED LOOPS — know the complexity! ──────────────────────
for (int i = 0; i < n; i++)          // ┐ O(n²) — all pairs
    for (int j = i+1; j < n; j++) {}  // ┘ j starts at i+1: n(n-1)/2 pairs

// ── BREAK / CONTINUE ─────────────────────────────────────────
break;    // exits the INNERMOST loop only
continue; // skips to NEXT iteration of innermost loop
// To break an outer loop: use labeled break
outer: for (...) {
    for (...) { if (found) break outer; } // exits outer loop!
}

☕ Java · Conditionals
// Standard if-else
if (x > 0) { ... }
else if (x == 0) { ... }
else { ... }

// Ternary — for simple single-value selection
int max = (a > b) ? a : b;

// Switch (Java 14+ — enhanced switch expression)
int days = switch (month) {
    case 1, 3, 5, 7, 8, 10, 12 -> 31;
    case 4, 6, 9, 11 -> 30;

};

// ⭐ Short-circuit evaluation — CRITICAL for null safety!
// && stops at first FALSE condition
if (node != null && node.val == target) { ... }
// if node==null, right side NEVER evaluates → no NullPointerException

// || stops at first TRUE condition
if (arr == null || arr.length == 0) return;
// if arr==null, second check NEVER runs → safe!

// ⚠️ Integer.MAX_VALUE and MIN_VALUE — essential constants

3.2 — Pass-by-Value: The Most Misunderstood Thing in Java

⚠️
Java is ALWAYS pass-by-value. No exceptions.
The confusion: for objects and arrays, the VALUE being passed is the reference (address).
A copy of the address is passed → mutations through that address persist. But reassigning the reference inside the method doesn't affect the caller.
☕ Java · Pass-by-Value Illustrated
// PRIMITIVE: copy of value — original unchanged
void tryChange(int x) { x = 99; }
int a = 5;
tryChange(a);
System.out.println(a);  // still 5 — x was a COPY

// ARRAY: copy of reference — mutations visible to caller!
void mutate(int[] arr) { arr[0] = 99; }
int[] arr = {1,2,3};
mutate(arr);
System.out.println(arr[0]);  // 99 — mutation persisted!

// REFERENCE REASSIGNMENT: doesn't affect caller
void reassign(int[] arr) { arr = new int[]{9,9,9}; }
reassign(arr);
System.out.println(arr[0]);  // still 99 — local arr was a copy of address

// ⭐ Consequence: swap(int a, int b) NEVER works
//   swap(int[] arr, int i, int j) DOES work (mutations on shared array)

4.4 — The equals() + hashCode() Contract

⚠️
The Contract: If a.equals(b) is true, then a.hashCode() == b.hashCode() MUST also be true.
Breaking this contract causes silent bugs in HashMap, HashSet, and every hash-based structure.
When using a custom object as a HashMap key, you MUST override both methods together.
☕ Java · Custom HashMap Key
class Point {
    int x, y;
    Point(int x, int y) { this.x = x; this.y = y; }

    @Override
    public boolean equals(Object o) {
        if (!(o instanceof Point)) return false;
        Point p = (Point) o;
        return x == p.x && y == p.y;
    }

    @Override
    public int hashCode() {
        return 31 * x + y;  // must be consistent with equals()
    }
}
// Without overriding: map.get(new Point(1,2)) → null even after put!
Map<Point, String> map = new HashMap<>();
map.put(new Point(1,2), "origin");
map.get(new Point(1,2));  // → "origin" ✓ (with both overrides)

⏱ Big-O Complexity Analysis

📌
What is Big-O?
Big-O is an upper bound on the growth rate of an algorithm's resource usage (time or space) as input size n → ∞. It describes how much slower your algorithm gets when you double the input — not the exact running time in milliseconds.

5.1 — The 4 Simplification Rules

# Rule Example Simplifies to
1 Drop constants O(3n + 500) O(n)
2 Drop non-dominant terms O(n² + n + log n) O(n²)
3 Different inputs → different variables Two arrays of size a and b processed sequentially O(a + b), NOT O(n)
4 Nested loops on same input → multiply Outer O(n), inner O(n) O(n²)

5.2 — Complexity Recognition by Code Pattern

☕ Java · Read Code → Know Complexity
// ── O(1) — fixed number of operations ────────────────────────
arr[i];                          // array access
map.get(key);                    // hashmap lookup
pq.peek();                       // heap peek

// ── O(log n) — input HALVED each step ────────────────────────
while (n > 1) { n /= 2; }      // log₂n iterations
// Binary search, heap operations, balanced BST ops

// ── O(√n) — loop to square root ──────────────────────────────
for (int i = 2; i*i <= n; i++) // prime check — √n iterations

// ── O(n) — single pass ───────────────────────────────────────
for (int i = 0; i < n; i++) {}  // n iterations
// Two-pointer, sliding window (both pointers move ≤ n times total)

// ── O(n log n) — sort, or divide + linear merge ──────────────
Arrays.sort(arr);               // O(n log n)
// Merge sort: log n levels × O(n) merge = O(n log n)

// ── O(n²) — nested loops over same input ─────────────────────
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) // always O(n²) even if j=i+1

// ── O(2ⁿ) — two recursive calls or all subsets ───────────────
// fib(n-1) + fib(n-2), or all 2ⁿ subsets of an array

// ── O(n!) — all permutations ─────────────────────────────────
// Generating all arrangements of n elements
Big-O n=10 n=100 n=1,000 n=10,000 Verdict
O(1) 1 1 1 1 🟢 Perfect
O(log n) 3 7 10 13 🟢 Excellent
O(√n) 3 10 32 100 🟢 Good
O(n) 10 100 1,000 10,000 🟢 Good
O(n log n) 33 664 10,000 133,000 🟡 Acceptable
O(n²) 100 10,000 1M 100M (slow) 🔴 Avoid for n>10⁴
O(2ⁿ) 1,024 10³⁰ 💀 — — 🔴 Only for n≤20
O(n!) 3.6M 💀 — — 🔴 Only for n≤10

5.3 — Space Complexity

📐
Always report BOTH time and space in interviews.
Then ask: "Can I reduce space to O(1) using in-place manipulation?"
Two-pointers, in-place reversal, and bit manipulation are classic O(1) space techniques.
☕ Java · Space Complexity Examples
// O(1) space — only fixed number of extra variables
int sum = 0;
for (int x : arr) sum += x;   // one variable, regardless of n

// O(n) space — proportional to input

// O(n) space — RECURSION! Each call frame is on the stack
int factorial(int n) {
    if(n==0) return 1;
    return n * factorial(n-1);  // depth n → O(n) stack space
}

🔤 String Essentials for DSA

Java strings are immutable — every modification creates a new string. This is the #1 interview trap for beginners.

🚨
DANGER: String concatenation in a loop is O(n²)!
String s = ""; for(int i=0; i<n; i++) s += "a";
Each += creates a new string and copies everything. Use StringBuilder instead.
StringBuilder sb = new StringBuilder(); for(...) sb.append("a"); return sb.toString();

String Operations You Must Know

Problem 01 · Brute → Optimize
Check if a Number is Prime
🔗 Related: LC 204 - Count Primes
Easy GoogleAmazonMicrosoft O(√n) Optimization
›
Problem

Given a positive integer n, return true if it is prime (divisible only by 1 and itself).

isPrime(2) → true isPrime(17) → true isPrime(15) → false (3 × 5 = 15) isPrime(1) → false (1 is NOT prime by definition)
Optimization insight

Factors come in pairs: if a × b = n, then min(a, b) ≤ √n. So we only need to check up to √n.

Additional: skip even numbers after checking 2 — halves the iterations.

▶Full solution with all 3 approaches + dry run
☕ Java · 3 Approaches
// Approach 1: Brute Force — O(n)
boolean isPrime_brute(int n) {
    if (n < 2) return false;
    for (int i = 2; i < n; i++)       // checks all the way to n-1
        if (n % i == 0) return false;
    return true;
}

// Approach 2: Optimized — O(√n)
boolean isPrime(int n) {
    if (n < 2) return false;
    if (n == 2) return true;          // 2 is prime
    if (n % 2 == 0) return false;     // all evens > 2 are not prime
    for (int i = 3; i*i <= n; i += 2) { // only odd i, stop at √n
        if (n % i == 0) return false;
    }
    return true;
}

// Dry run: isPrime(37)
// n=37, not even. i=3: 9≤37, 37%3≠0 | i=5: 25≤37, 37%5≠0
// i=7: 49>37 → STOP → return true ✓
// Only 3 iterations vs 35 in brute force!
Brute TimeO(n)
Optimal TimeO(√n)
SpaceO(1)
Problem 02 · Two Pointers Preview
Reverse an Array In-Place
🔗 LC 344 - Reverse String
Easy AmazonMicrosoftFacebook Two Pointers
›
Problem

Reverse an array in-place using O(1) extra space. Do not create a new array.

Input: [1, 2, 3, 4, 5] → Output: [5, 4, 3, 2, 1] Input: [1, 2] → Output: [2, 1]
▶Solution + Dry Run
☕ Java
void reverse(int[] arr) {
    int left = 0, right = arr.length - 1;
    while (left < right) {
        int tmp  = arr[left];  // 3-step swap
        arr[left]  = arr[right];
        arr[right] = tmp;
        left++;   right--;     // converge inward
    }
}
// Dry run: [1,2,3,4,5]
// left=0,right=4: swap(1,5) → [5,2,3,4,1]
// left=1,right=3: swap(2,4) → [5,4,3,2,1]
// left=2,right=2: left==right → stop ✓
// n/2 = 2 swaps for n=5. O(n) time, O(1) space.
TimeO(n)
SpaceO(1)
Problem 03 · Binary Search Template
Search in Sorted Array
🔗 LC 704 - Binary Search
Easy GoogleAmazonMicrosoft Binary Search LC #704
›
Problem

Given a sorted array and a target, return the index of target, or -1 if not found. Must run in O(log n).

arr=[1,3,5,7,9,11,13], target=7 → index 3 arr=[1,3,5,7,9,11,13], target=6 → -1
▶Solution + Overflow-safe mid calculation
☕ Java
int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;  // ← Safe! Avoids int overflow
        // Buggy version: (lo + hi) / 2 can overflow when lo+hi > 2.1B!
        if      (arr[mid] == target) return mid;
        else if (arr[mid] < target)  lo = mid + 1;
        else                         hi = mid - 1;
    }
    return -1;
}
// Dry run: arr=[1,3,5,7,9], target=7
// lo=0,hi=4 → mid=2 → arr[2]=5 < 7 → lo=3
// lo=3,hi=4 → mid=3 → arr[3]=7 == 7 → return 3 ✓
TimeO(log n)
SpaceO(1)
🔑
Template to memorize: The binary search template above is the foundation for 30+ harder problems. The key variants: find first occurrence (hi = mid when found), find last occurrence (lo = mid when found), binary search on answer (apply binary search on the answer space, not array).
Problem 04 · HashMap Pattern
Two Sum
🔗 LC 1 - Two Sum
Easy GoogleAmazonFacebook MicrosoftAppleUber Complement Lookup LC #1
›
Problem

Given an array and target, return indices of two numbers that add to target. Exactly one solution exists.

Input: nums=[2,7,11,15], target=9 → Output: [0,1] Input: nums=[3,2,4], target=6 → Output: [1,2]
▶Brute O(n²) → Optimal O(n) with dry run
☕ Java · Both Approaches
// Brute force: O(n²) — check all pairs
int[] twoSum_brute(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++)
        for (int j = i+1; j < nums.length; j++)
            if (nums[i] + nums[j] == target)
                return new int[]{i, j};
    return new int[]{};
}

// Optimal: O(n) — complement lookup in HashMap
int[] twoSum(int[] nums, int target) {
    Map<Integer,Integer> seen = new HashMap<>(); // value → index
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (seen.containsKey(complement))
            return new int[]{seen.get(complement), i};
        seen.put(nums[i], i);  // store AFTER checking (prevents self-pair)
    }
    return new int[]{};
}

// Dry run: nums=[2,7,11,15], target=9
// i=0: complement=7, seen={} → not found → seen={2:0}
// i=1: complement=2, seen={2:0} → FOUND! → return [0,1] ✓
Brute TimeO(n²)
Optimal TimeO(n)
SpaceO(n)
Problem 05 · Complexity Analysis
Maximum Subarray Sum (Kadane's Algorithm)
🔗 LC 53 - Maximum Subarray
Medium GoogleAmazonFacebook MicrosoftApple Kadane's Algorithm LC #53
›
Problem

Find the contiguous subarray with the largest sum.

Input: [-2, 1, -3, 4, -1, 2, 1, -5, 4] Output: 6 (subarray [4, -1, 2, 1])
Kadane's Key Insight

At each position i, the max subarray ending at i is either the current element alone (start fresh) or the current element extended from the previous best. We never carry a negative prefix.

▶Solution with full Kadane's derivation
☕ Java · Kadane's Algorithm
int maxSubArray(int[] nums) {
    int maxSum = nums[0];     // global max (handles all-negative arrays)
    int curSum = nums[0];     // max ending at current position

    for (int i = 1; i < nums.length; i++) {
        // Either extend current subarray, or start fresh
        curSum = Math.max(nums[i], curSum + nums[i]);
        maxSum = Math.max(maxSum, curSum);
    }
    return maxSum;
}

// Trace: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
// i=1: curSum=max(1, -2+1=-1)=1,    maxSum=1
// i=2: curSum=max(-3, 1-3=-2)=-2,   maxSum=1
// i=3: curSum=max(4, -2+4=2)=4,     maxSum=4
// i=4: curSum=max(-1, 4-1=3)=3,     maxSum=4
// i=5: curSum=max(2, 3+2=5)=5,      maxSum=5
// i=6: curSum=max(1, 5+1=6)=6,      maxSum=6 ← answer
// i=7: curSum=max(-5, 6-5=1)=1,     maxSum=6
// i=8: curSum=max(4, 1+4=5)=5,      maxSum=6
// Return 6 ✓
TimeO(n)
SpaceO(1)
Problem 06 · HashSet Pattern
Contains Duplicate
🔗 LC 217 - Contains Duplicate
Easy AmazonGoogleMicrosoft HashSet
›
Problem

Given an integer array nums, return true if any value appears at least twice in the array. Return false if every element is distinct.

Input: [1, 2, 3, 1] → Output: true (1 appears twice) Input: [1, 2, 3, 4] → Output: false (all unique) Input: [1, 1, 1, 3, 3] → Output: true
Think Step by Step

Why this problem matters: It teaches you the most fundamental pattern in DSA — using a HashSet for O(1) lookups. You'll use this pattern in 50+ problems later.

Three approaches, from naive to smart:

  1. Brute force: Check every pair (i, j) — if nums[i] == nums[j] → true. Time: O(n²). Too slow.
  2. Sort first: Duplicates become adjacent. Sort → scan for adjacent equals. Time: O(n log n).
  3. HashSet (optimal): Walk once. For each number, check if it's already in the set. If yes → return true. Else add it. Time: O(n), Space: O(n).
▶Full solution with dry run
☕ Java · HashSet Approach
boolean containsDuplicate(int[] nums) {
    Set<Integer> seen = new HashSet<>();
    for (int num : nums) {
        if (seen.contains(num)) return true;  // already seen!
        seen.add(num);                         // remember it
    }
    return false;  // no duplicates found
}

// Dry run: [1, 2, 3, 1]
// i=0: seen={}, 1 not in seen → add 1 → seen={1}
// i=1: seen={1}, 2 not in seen → add 2 → seen={1,2}
// i=2: seen={1,2}, 3 not in seen → add 3 → seen={1,2,3}
// i=3: seen={1,2,3}, 1 IS in seen → return true ✓
TimeO(n)
SpaceO(n)
💡
Pattern to memorize: Whenever you need to check "have I seen this before?", use a HashSet. O(1) lookup vs O(n) linear scan. This is the #1 optimization technique in interviews.
Problem 07 · Frequency Counting
Valid Anagram
🔗 LC 242 - Valid Anagram
Easy AmazonGoogleUber Frequency Count
›
Problem

Given two strings s and t, return true if t is an anagram of s (same letters, same counts, different order).

isAnagram("anagram", "nagaram") → true isAnagram("rat", "car") → false isAnagram("listen", "silent") → true
Key Insight

Two strings are anagrams if and only if they have the same character frequencies. This is the frequency counting pattern — you'll use it everywhere (group anagrams, permutation checking, hash comparisons).

▶Full solution with 2 approaches
☕ Java · Frequency Array
// Approach 1: Sort and compare — O(n log n)
boolean isAnagram_sort(String s, String t) {
    if (s.length() != t.length()) return false;
    char[] a = s.toCharArray(), b = t.toCharArray();
    Arrays.sort(a);
    Arrays.sort(b);
    return Arrays.equals(a, b);
}

// Approach 2: Frequency count — O(n) time, O(1) space
boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;
    int[] freq = new int[26];  // 26 lowercase letters
    for (int i = 0; i < s.length(); i++) {
        freq[s.charAt(i) - 'a']++;  // increment for s
        freq[t.charAt(i) - 'a']--;  // decrement for t
    }
    for (int count : freq)
        if (count != 0) return false;
    return true;
}

// Dry run: isAnagram("cat", "tac")
// freq = [0]*26. Process each pair:
// i=0: 'c' → freq[2]++ = 1,  't' → freq[19]-- = -1
// i=1: 'a' → freq[0]++ = 1,  'a' → freq[0]-- = 0
// i=2: 't' → freq[19]++ = 0, 'c' → freq[2]-- = 0
// All freq[] == 0 → return true ✓
TimeO(n)
SpaceO(1)
🔮
Why char - 'a'? In ASCII, 'a'=97, 'b'=98, etc. So 'c' - 'a' = 2. This maps any lowercase letter to an index 0–25. You'll use this trick in almost every string problem.
Problem 08 · Prefix Sum Introduction
Running Sum of 1D Array
🔗 LC 1480 - Running Sum
Easy Amazon Prefix Sum
›
Problem

Given an array nums, return the running sum: runningSum[i] = sum(nums[0]...nums[i]).

Input: [1, 2, 3, 4] → Output: [1, 3, 6, 10] Input: [3, 1, 2, 10, 1] → Output: [3, 4, 6, 16, 17]
Why this matters (a lot)

This is your first encounter with the prefix sum pattern. Prefix sums allow you to compute the sum of ANY subarray in O(1). This technique appears in:

  • Range sum queries (LC 303)
  • Subarray sum equals K (LC 560) — uses prefix sum + HashMap
  • Product of array except self (LC 238)
  • Many DP problems that optimize with prefix/suffix arrays
▶Full solution with dry run
☕ Java · In-place
int[] runningSum(int[] nums) {
    for (int i = 1; i < nums.length; i++) {
        nums[i] += nums[i - 1];  // accumulate
    }
    return nums;
}

// Dry run: [1, 2, 3, 4]
// i=1: nums[1] = 2 + 1 = 3   → [1, 3, 3, 4]
// i=2: nums[2] = 3 + 3 = 6   → [1, 3, 6, 4]
// i=3: nums[3] = 4 + 6 = 10  → [1, 3, 6, 10] ✓

// 🎯 Bonus: How to get sum of subarray [i..j]?
// sum(i, j) = prefix[j] - prefix[i-1]
// Example: sum of nums[1..2] = prefix[2] - prefix[0] = 6 - 1 = 5
// Verify: nums[1] + nums[2] = 2 + 3 = 5 ✓
TimeO(n)
SpaceO(1)
Problem 09 · Bit Manipulation Preview
Single Number (XOR Trick)
🔗 LC 136 - Single Number
Easy AmazonGoogleApple XOR / Bit Manipulation
›
Problem

In an array where every element appears twice except one, find the single element. Must use O(1) extra space.

Input: [2, 2, 1] → Output: 1 Input: [4, 1, 2, 1, 2] → Output: 4
The XOR Trick — Master This

XOR (exclusive or) has two magical properties:

  • a ^ a = 0 — any number XOR'd with itself is zero
  • a ^ 0 = a — any number XOR'd with zero is itself
  • XOR is commutative and associative — order doesn't matter

So if you XOR all numbers: 2 ^ 2 ^ 1 = 0 ^ 1 = 1. Duplicates cancel out!

▶Full solution with dry run
☕ Java · XOR
int singleNumber(int[] nums) {
    int result = 0;
    for (int num : nums) {
        result ^= num;  // XOR with each element
    }
    return result;
}

// Dry run: [4, 1, 2, 1, 2]
// result = 0
// 0 ^ 4 = 4  (binary: 000 ^ 100 = 100)
// 4 ^ 1 = 5  (binary: 100 ^ 001 = 101)
// 5 ^ 2 = 7  (binary: 101 ^ 010 = 111)
// 7 ^ 1 = 6  (binary: 111 ^ 001 = 110)  ← 1 cancels
// 6 ^ 2 = 4  (binary: 110 ^ 010 = 100)  ← 2 cancels
// Return 4 ✓
TimeO(n)
SpaceO(1)
🧠
Where else does XOR appear? Missing Number (LC 268), Find Duplicate (LC 287), Single Number II (LC 137), Single Number III (LC 260). Understanding XOR deeply sets you up for Lecture 3: Bit Manipulation.
Problem 10 · Greedy / One-Pass
Best Time to Buy and Sell Stock
🔗 LC 121 - Best Time to Buy and Sell Stock
Easy AmazonFacebookGoogle Track Min/Max
›
Problem

You have an array prices where prices[i] is the stock price on day i. Choose a single day to buy and a later day to sell. Maximize profit. If no profit is possible, return 0.

Input: [7, 1, 5, 3, 6, 4] → Output: 5 (buy at 1, sell at 6) Input: [7, 6, 4, 3, 1] → Output: 0 (prices only fall)
Think Step by Step

Brute force: Check every (buy, sell) pair where buy < sell → O(n²).

Key insight: As you scan left to right, track the minimum price seen so far. At each day, the best you can do is price[i] - minPrice. Track the maximum of these profits.

This is the "track min/max so far" pattern — appears in many problems!

▶Full solution with dry run
☕ Java · One Pass
int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE;
    int maxProfit = 0;

    for (int price : prices) {
        if (price < minPrice) {
            minPrice = price;            // found a cheaper buying day
        } else {
            maxProfit = Math.max(maxProfit, price - minPrice);  // check profit
        }
    }
    return maxProfit;
}

// Dry run: [7, 1, 5, 3, 6, 4]
// day 0: price=7, minPrice=7, maxProfit=0
// day 1: price=1 < 7 → minPrice=1
// day 2: price=5, profit=5-1=4 → maxProfit=4
// day 3: price=3, profit=3-1=2 → maxProfit=4 (no change)
// day 4: price=6, profit=6-1=5 → maxProfit=5 ← best!
// day 5: price=4, profit=4-1=3 → maxProfit=5 (no change)
// Return 5 ✓
TimeO(n)
SpaceO(1)
